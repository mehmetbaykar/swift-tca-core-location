# Location Manager

<img width="1290" alt="image" src="https://user-images.githubusercontent.com/135203/82390225-0187c880-99f3-11ea-8ae7-e33f6993f89d.png">

This application demonstrates a shared TCA feature running on iOS and macOS with Core Location and MapKit. It displays a map, searches for points of interest by category, and can center the map on the user's current location after authorization.

## Structure

The example is generated by Tuist from [../Project.swift](../Project.swift). [../Tuist/Package.swift](../Tuist/Package.swift) points to the root package with `.package(path: "../..")`, so the example uses the local `ComposableCoreLocation` target instead of a separate checkout.

* `Common` contains the shared `LocationManagerFeature` framework.
* `Common/AppCore.swift` contains the `@Reducer`, `@ObservableState`, actions, dependencies, and effects.
* `Mobile` contains the iOS app shell.
* `Desktop` contains the macOS app shell.
* `CommonTests` contains Swift Testing coverage for the shared feature.

## Core Logic

The feature uses:

* `@Dependency(\.locationManager)` for Core Location.
* `@Dependency(\.localSearch)` for MapKit local search.
* `LocationManager.delegate()` as an `AsyncStream` of delegate actions.
* Async `.run` effects for permission requests, location requests, and searches.
* `@Presents` and `.ifLet` for alert presentation.

The reducer owns the cross-platform behavior once. The iOS and macOS targets only provide platform-specific shells around that shared feature.

## Tests

The tests use Swift Testing with TCA's `TestStore`. `LocationManager.failing` is used by default, and each test overrides only the dependency endpoints it needs.

From the repository root:

```sh
make generate-examples
make test-examples
```
